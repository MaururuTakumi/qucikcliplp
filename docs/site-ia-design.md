## 0. 設計の背骨（3行）

1. **LayerXとhonkomaは「コンバージョンの正体」が違う。** LayerXのコーポサイトは販売を製品サイトに逃がせるので採用が最上位CV。honkomaは**コーポサイトそのものが営業装置**＝最上位CVは商談（ご相談/AI診断リード）、第二CVが採用。ピル（常時CTA）は商談に、ナビの一等地は採用に、と**CTAの階層で2オーディエンスを両立**させる。
2. **2レンズ運用**: 全セクション・全導線を「👤顧客（格・信頼・商談化）」「🔥採用（若手AIマフィアを惹く）」の2レンズで設計。LayerX同様「**上半分=顧客向け、下半分=採用向け**」の縦割りを基本にしつつ、honkomaでは**マニフェストと"人"が両レンズを同時に打つ共有資産**（顧客には「本気の会社」、候補者には「俺の居場所」に見える）。
3. **情報は減らす。** 10名の会社が大企業のIAを真似ると空洞が見える。ページ数を絞り（11→9ルート）、薄い縦割りLP（/d2c /review-ai）は捨て、残したページを深くする。

## 1. 推奨サイトマップ + グローバルナビ仕様

### 1.1 サイトマップ（図）

```
/                        ホーム（予告編・2オーディエンス縦割り）
├─ /about                私たちについて（ミッション / 行動指針 / 会社概要 を1ページ集約）
├─ /team                 メンバー（若きAIマフィア・顔写真グリッド）※刷新済み・維持
├─ /product              サービス（二層オファーの整理: 3つのかたち × 3プラン × 業界特化）
│   └─ /hotel            ホテル向け Revenue Intelligence（唯一の業界特化LP・残す）
├─ /case-studies         導入事例（信頼エンジン・3層開示）
├─ /recruit              採用（新設・「自分事」+ 林ストーリー軸）

└─ /privacy              プライバシーポリシー

削除（リダイレクト）: /d2c → /product#solutions ／ /review-ai → /product#solutions
```

- **9ルート構成**（現11ルート）。LayerX原則1「浅いナビ」を「浅いサイト」まで拡張。10名の実体に対しページ数が信頼を毀損しない上限がこのあたり。
- `/about` と `/team` はURLを分けたまま、**ナビ上は「私たちについて▾」に束ねる**。LayerXは会社系5コンテンツを1ページ+タブに集約しているが、honkomaの会社系は実質2.5ページ分しかない。**タブ付きメガページを新造するより、刷新済み資産（About/Team）を活かしてナビ側で集約する方が、工数対効果と保守性で勝る**（原則3を「ページ集約」でなく「ナビ集約+Aboutの拡張」で満たす）。
- `/product` のURLは維持（改名リダイレクトの負債を作らない）。ラベルは「サービス」。

### 1.2 グローバルナビ仕様（Header.tsx の NAV / OVERLAY_LINKS / CTA を差し替え）

```
ロゴ | 私たちについて▾ | サービス▾ | 導入事例 | 採用情報 | [AIに聞いてみる outline] [ご相談はこちら fill]
       ├ ミッション・行動指針 (/about)      ├ サービス一覧 (/product)
       ├ 会社概要 (/about#profile)          └ ホテル向け (/hotel)
       └ メンバー (/team)
```

| 要素 | 仕様 | レンズ | 判断根拠（B2Bコンサル視点） |
|---|---|---|---|
| トップ項目 | **4つだけ**（私たちについて/サービス/導入事例/採用情報）。「ホーム」はロゴに担わせ削除 | — | LayerX原則1。5項目のLayerXよりさらに浅く。B2Bの訪問者は平均2〜3ページしか見ない — 迷わせない |

| 導入事例 | トップ項目に昇格（現在は outline ピル） | 👤 | B2Bの検討者が最初に探すのは事例。LayerXの「ニュース」枠のhonkoma的等価物（原則5）。ピルより恒常的な一等地へ |
| 採用情報 | トップ項目として独立（ドロップダウンなし・/recruit 直リンク） | 🔥 | 原則4「採用を第一級に」。ただしピルは商談に譲る（§0-1のCTA階層）。ナビ右端=視線の終点に置き、候補者には必ず見える |

| ピル(fill) | 「ご相談はこちら」→ /contact（現状維持） | 👤CV | 「もう決めた人」の最短経路。チャットと二段構え（#38 §1.3の上流/下流と一致） |

**採用ピルを置かない理由（明示）**: LayerXが右端ピルを2本とも採用に使うのは、コーポサイトのCVが採用だから。honkomaで同じことをすると、**商談の最短経路が消えて営業装置としてのLPが死ぬ**。採用はナビ一等地 + トップページJoin Usセクション + フッターで三重露出させれば、候補者に対する「第一級扱い」は十分に伝わる。

## 2. トップページ最終セクション構成

現状（Hero / 事業3枚 / 提供価値 / 実績厳選 / Our Mission / CTA）の評価: **上半分は完成度が高い。問題は下半分** — ①Our Mission が Hero のマニフェストと役割重複し、1ページに理想の再掲が2回ある割に「人」が出てこない ②採用の入口がゼロ（LPの2本柱の片方が不在） ③AIチャット起動点（#38 A/C/D）が未配置。

| # | セクション | 役割 | レンズ | 既存資産 / 変更 |

**「採用が会社の顔と両立する見せ方」の核心（#6の設計意図）**: 採用コンテンツを「求人情報」として置くと会社の顔を毀損するが、**「実在する10人の顔と熱」として置くと逆に格を上げる**。B2Bの発注者は「誰がやるのか」を最後に必ず見る — チーム写真マソンリーは候補者には「仲間の顔」、顧客には「実行部隊の実在証明」として同じ1セクションで二重に働く。だから Join Us はトップから外さない・隠さない。LayerXが濃紺セクションで採用だけ色を切り替えるのと同様、**#6は inverse で「ここから体温が変わる」ことを視覚宣言**する。

**ページ全体の2レンズ配分**: #1両立 → #2〜#5顧客 → #6採用 → #7顧客、の「顧客サンドイッチ」。候補者は#1で掴んで#6で受け、顧客は#2で即CV・#7で締める。LayerXの上下二分構造の10名企業版。

## 4. /recruit 採用ページ（新設）

前提: **募集職種の実体が薄い** → 求人票の空欄で深さを偽装しない。「職種で募らず、人物像で募る」オープンポジション型に振り切る。深さは職種数でなく**物語の解像度**で出す。
