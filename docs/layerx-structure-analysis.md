## 0. 骨子（1行）
**「浅く強いグローバルナビ＋トップは全体のティーザー、深さは下層ページへ」**。トップで会社の全貌を"予告編"として見せ、詳細（事業・会社・採用）は専用ページに逃がす。採用を第一級市民として独立させている。

## 1. グローバルナビ / サイトマップ（IA）
```
ロゴ | 私たちについて▾ | 事業内容 | ニュース | お問い合わせ | 採用情報▾ | [Join Us] [採用情報]
        ├ ミッション                                          （右端に2ピル）
        ├ 行動指針
        ├ 会社概要
        ├ 経営メンバー
        └ DE&Iポリシー
```
- **トップ項目は5つだけ**（私たちについて/事業内容/ニュース/お問い合わせ/採用情報）＋2つはドロップダウン。フラットに項目を並べない。
- **「私たちについて」に会社系を全部束ねる**（ミッション/行動指針/会社概要/経営メンバー/DE&I）。
- **採用が独立**（専用ドロップダウン＋右端に「採用情報」塗りピル＋別サブドメイン jobs.layerx.co.jp）。＝採用を最重要導線として扱う。
- スクロールで**Menuピルにモーフ**（全ページ共通）。

## 2. トップページの構成（縦の"予告編"）
```
① Hero        ミッション大見出し（縦書き相当）＋英字＋CTA「私たちについて」＋スクロール連動ビジュアル
② What We Do  事業3枚（各1行タグライン＋→）。CTA「事業内容へ」
③ Latest News 日付＋カテゴリピル＋ロゴ＋見出しの行リスト数件。CTA「ニュース一覧へ」
④ Join Us     ミッション再掲＋大円CTA「採用情報へ」＋チーム写真マソンリー（カルチャー前面）
⑤ Footer      多列（会社/事業/採用・問い合わせ/法務/SNS/ISO）
```
**設計思想**: トップは各領域の"入口"だけ。事業も会社も採用もニュースも「さわり」を見せて、深追いは下層へ誘導する。1画面1メッセージ・余白広め。

## 3. 下層ページの構成パターン
- **私たちについて（会社系の集約ページ）**: ページ上部に**タブ**（ミッション/行動指針/会社概要/経営メンバー/DE&I）。同一ページ内を縦スクロールで:
  - ミッション再掲（大見出し）
  - 行動指針（Our Principles を **01/05〜05/05 のナンバリング**で1原則1画面＋各原則に専用3Dビジュアル）
  - 会社概要（ミッション再掲＋代表者署名＋Company Deckへの導線）
  - 経営メンバー（顔写真グリッド）
  → **"会社を1ページで語り切る"集約**。会社概要すら理念の再確認の場。
- **事業内容**: 3事業の詳細（各事業に個別の深掘り）。
- **ニュース**: 行リストの一覧＋カテゴリ。信頼性・活動量の証明。
- **採用（別サブドメイン）**: 旗コピー→感情→協働→大義の3段→**3大分類×10職種のカスタマイズ選択**→カルチャー/働き方。

## 4. フッターの構成（多列IA）
私たちについて（ミッション/行動指針/会社概要/経営メンバー/DE&I）｜事業内容（3事業）｜採用/お問い合わせ｜法務スタック（プレスキット/各ポリシー/情報セキュリティ/外部送信規律/反社/カスハラ/イベント行動規範）｜SNS（X/note/SpeakerDeck/GitHub/Facebook）｜© ＋タグライン｜ISO27001バッジ。
→ フッターが**サイト全体のインデックス＋信頼担保（法務・認証）**を兼ねる。

## 5. 構成上の「効いている」判断（honkomaが真似るべき原則）
1. **浅いナビ**: トップ5項目＋ドロップダウン。項目を増やさない。
2. **トップ=予告編**: 各領域の入口だけ。深さは下層へ。事例やサービスを縦に大量に積まない。
3. **会社系を1ページに集約**（タブ）。理念を軸に。
4. **採用を第一級に**（独立ナビ＋ピル）。
5. **ニュース/実績で信頼**（活動量の可視化）。
6. **フッターがサイトの地図＋信頼担保**。
7. 全ページ通じて **1画面1メッセージ・余白・静かな確信・煽らない**。

## 6. honkomaとの差分（構成設計で考慮すべき点 — fableへの論点）
- **規模/リソース**: honkomaは10名。採用サブドメインや大量ニュースは持てない → 採用/ニュースをどう軽量に扱うか。
- **オファーの二層性**: honkomaは①会社レベルの提供（AI導入/FDE/プロダクト開発）と②プロダクト化したもの（AI秘書/顧問/ホテル向け/D2C/口コミAI）を両方持つ。LayerXの"3事業"より複雑 → サービスの階層化が要る。
- **既存の下層ページ**: /product・/hotel・/d2c・/review-ai・/case-studies が縦割りで存在 → LayerX式に束ねるか、残すか。
- **採用の実体**: 「自分事」文化＋林ストーリーはあるが、募集職種の実体は？ → 採用ページの深さをどうするか。
- **AIチャット動線(#38)**: honkoma固有の"AIに聞いてみる"導線をこの構成のどこに差すか。
- **現状のヘッダー**: 既に「ホーム/会社概要/サービス▾/メンバー」に整理済み → LayerXの「私たちについて集約」型に寄せるか、現状維持か。
