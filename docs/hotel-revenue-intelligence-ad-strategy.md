# Hotel Revenue Intelligence 広告・獲得戦略

## 結論

初期は広く広告を打たない。  
まずは「福岡・九州のホテル運営会社 / 複数施設を持つ本部 / Revenue担当」に絞り、ABM型の営業と高意図検索広告を組み合わせる。

LPの役割は、広告で完結させることではなく、初回接触後に「この会社はホテル収益運用の話ができる」と判断してもらうこと。

## ポジショニング

競合カテゴリは、Rate Shopping / Revenue Management / Market Intelligence。
ただし初期の勝ち筋は、Lighthouse等と機能表で正面比較することではない。

打ち出しは以下に寄せる。

- イベント需要、競合料金、販売条件の変化を収益会議の打ち手に変える
- 通知だけではなく、確認・共有・対応完了までをワークフロー化する
- 日本のホテル運営会社に合わせ、競合セット・会議運用・通知先まで設計する
- 外部市場データから小さく始め、必要に応じて自館データ連携を拡張する

避ける訴求:

- データ取得方法を前面に出す
- PMS未接続を売りにする
- 単なる価格比較ツールに見せる
- 成果を断定する

## 初期ターゲット

### Tier 1: 複数施設運営会社

狙う理由:

- 1契約あたりの施設数が増えやすい
- 本部Revenue / 営業企画の課題が明確
- 施設横断の判断基準を揃える価値が出やすい

初回訴求:

> 複数施設のイベント需要・競合料金変化を、収益会議で使える通知と対応履歴にします。

### Tier 2: 都市型ホテル / 駅前ホテル

狙う理由:

- 競合セットが明確
- イベント・曜日・連休の影響を受けやすい
- OTA担当とRevenue担当が課題を持ちやすい

初回訴求:

> 博多駅前のような競争エリアで、確認すべき日付と料金変化を見逃しにくい運用にします。

### Tier 3: 独立系ホテル

狙う理由:

- 手作業確認の負担が大きい
- 専任Revenue担当がいない場合に刺さる

初回訴求:

> 専任担当がいなくても、見るべき日付・競合変化・対応状況をチームで確認できます。

## チャネル戦略

### 1. ABMアウトバウンド

最優先。広告より先に実施する。

対象:

- ホテル運営会社の代表 / 役員
- 本部Revenue / 営業企画
- GM / 支配人
- 宿泊販売 / OTA担当

導線:

1. 対象会社ごとに、運営ブランド・施設数・主要エリアを整理
2. その会社向けに「イベント需要 / 競合料金 / 施設横断」のどれが刺さるか仮説化
3. 1社1枚の簡易提案メモを作る
4. LinkedIn / 問い合わせ / 電話で接触
5. LPは信頼補強として送る

KPI:

- 50社リスト化
- 150接点
- 初回返信率 5-10%
- 商談化率 2-5%

### 2. Google検索広告

目的は刈り取り。配信量は少なくてよい。

想定キーワード:

- ホテル レベニューマネジメント ツール
- ホテル 料金管理 システム
- ホテル 競合 価格 調査
- ホテル イベント 需要 料金
- ホテル revenue management 日本
- ホテル rate shopper

広告文方向:

- ホテル向けRevenue Intelligence
- イベント需要・競合料金を通知
- 収益会議で使える判断材料を整理
- 複数施設の料金判断を同じ見方に

除外したい検索意図:

- 宿泊予約したい一般消費者
- ホテル料金の一般比較
- 旅行者向け検索

KPI:

- 指名外CTR 3%以上
- LP CVR 2%以上
- 商談化CPAの上限を営業単価から逆算

### 3. LinkedIn広告

広く配信すると高くなるため、ABM補助に限定する。

用途:

- 接触済み企業へのリターゲティング
- ホテル運営会社 / レベニュー職種への認知補助
- 営業メール後の信頼形成

広告クリエイティブ:

- 「イベント需要を、料金判断の通知に」
- 「競合料金の変化を、収益会議の打ち手へ」
- 「複数施設のRevenue判断を、同じ見方に」

### 4. 業界接点

広告より商談化しやすい可能性がある。

- ホテル業界メディアへの寄稿
- レベニューマネジメント勉強会
- 宿泊業向け展示会・商談会
- PMS / サイトコントローラー / 予約管理会社との共催ウェビナー

テーマ例:

- 「イベント需要期に、料金判断が遅れる理由」
- 「複数施設運営でRevenue判断を揃える方法」
- 「競合料金チェックを会議資料で終わらせない運用」

## 90日実行プラン

### 0-30日

- 福岡・九州の対象会社50社を作る
- 役職別メッセージを3パターン作る
- LPへの流入をABM中心で開始
- Google検索広告は小額で高意図キーワードのみ開始

### 31-60日

- 返信率・商談化率でメッセージを修正
- LPに「本部向け」「GM向け」「Revenue向け」の導線を追加検討
- 商談で刺さった言葉を広告見出しへ反映

### 61-90日

- 商談化した業態に絞り、広告予算を寄せる
- 業界メディア / セミナー / パートナー接点を開始
- 初期PoCの導入事例を匿名でもよいので作る

## LP改善候補

優先順:

1. 本部・複数施設向けセクションの強化
2. 役職別CTAの追加
3. イベント需要アラートの具体例追加
4. 匿名ケーススタディ追加
5. Google広告用の短い専用LP作成

## 営業での一言

> 価格比較ツールではなく、イベント需要・競合料金・対応状況を、収益会議で動かせる形にする仕組みです。
