# Hotel Revenue Intelligence Outreach Kit

## 役職別メッセージ

### 1. Owner / Executive

件名:

- 周辺イベントと競合料金の変化、日々確認できていますか
- ホテル収益運用の判断速度を上げるご提案
- 複数施設のRevenue判断を揃えるためのご相談

本文:

```
突然のご連絡失礼いたします。株式会社honkomaの林です。

ホテル向けに、イベント需要・競合料金の変化・対応状況をひとつの流れで管理するRevenue Intelligence基盤を開発しています。

経営側から見ると、料金変更そのものよりも、
「需要が動いた日をチームが早く把握できているか」
「競合の価格変化に対して、誰が何を確認したか」
「施設ごとの判断が属人化していないか」
が課題になりやすいと考えています。

貴社の対象エリア・比較対象施設・会議フローに合わせて、どの変化を通知すべきかを15分ほどで壁打ちできればと思いご連絡しました。

一度、貴社での活用可能性を短時間で確認させていただけないでしょうか。
```

電話での入り:

```
ホテルの料金判断や競合料金確認を、イベント需要や対応履歴と一緒に管理する仕組みを作っています。
営業資料の送付ではなく、まず貴社で見るべき日付や比較対象があるかを確認したく、ご担当の方におつなぎいただくことは可能でしょうか。
```

### 2. GM / 宿泊支配人

件名:

- 料金判断が一部担当者に寄っていないかのご相談
- イベント需要時の料金確認フローについて
- 収益会議で見るべき変化を整理するご提案

本文:

```
突然のご連絡失礼いたします。株式会社honkomaの林です。

ホテルのGM・支配人向けに、需要変化、競合料金、対応状況を収益会議で使える形に整理するRevenue Intelligence基盤を開発しています。

多くのホテルでは、イベント日程や競合料金の確認を担当者が個別に行い、
会議時点で「どの日付を見直すべきか」「誰が確認したか」が残りにくい状態になりがちです。

HASIPでは、確認すべき日付、競合料金の変化、料金・在庫・販路の対応候補をチームで共有できる形にします。

貴館の現在の料金確認フローに対して、どこにアラートや対応履歴を入れると使いやすいか、15分ほどでご相談できないでしょうか。
```

### 3. Revenue Manager

件名:

- 日次の料金確認をアラート駆動にできないかのご相談
- 競合料金の変化と需要シグナルを一緒に見る仕組み
- レベニュー判断の手作業を減らすご提案

本文:

```
突然のご連絡失礼いたします。株式会社honkomaの林です。

Revenue担当者向けに、イベント需要、競合料金の変化、料金機会、対応履歴をひとつのワークフローで扱う仕組みを開発しています。

置き換えたいのはRevenue担当者の判断ではなく、その前段にある手作業の確認です。

毎日すべてを見るのではなく、
「どの日付を見るべきか」
「どの競合変化に注意すべきか」
「誰に共有し、どの対応を検討するか」
をアラートとして整理します。

現在の日次確認フローと比較して、アラート化できる部分があるかを短時間で壁打ちさせてください。
```

### 4. 宿泊販売 / OTA担当

件名:

- イベント需要時のOTA施策と料金判断の連携について
- 販路確認と料金判断を同じ画面で追うご提案
- 競合料金変化をOTA運用に反映する仕組み

本文:

```
突然のご連絡失礼いたします。株式会社honkomaの林です。

ホテルの宿泊販売・OTA担当向けに、需要変化、競合料金、販路確認、対応状況をチームで共有する仕組みを作っています。

イベントや連休で需要が変わった時、料金、在庫、販路、パリティ確認が別々に進むと、対応の抜けや遅れが起きやすくなります。

HASIPでは、確認すべき日付と変化を通知し、担当者への共有、料金見直し、販路確認、対応完了までを一連の流れとして扱います。

貴館のOTA運用に照らして、どの変化を通知できると使いやすいかを15分ほどで伺えないでしょうか。
```

### 5. Corporate HQ / Area Manager

件名:

- 複数施設のRevenue判断を同じ見方にするご提案
- 施設ごとの料金判断の属人化を減らす仕組み
- 本部から見たイベント需要・競合料金の管理について

本文:

```
突然のご連絡失礼いたします。株式会社honkomaの林です。

複数施設を運営するホテル会社向けに、需要変化、競合料金、料金機会、対応状況を施設横断で確認できるRevenue Intelligence基盤を開発しています。

複数施設になるほど、施設ごとに比較対象、確認タイミング、料金判断の基準がずれやすくなります。

HASIPでは、施設ごとの裁量を残しながら、
「どの施設で、どの日付に、どの変化が起きているか」
「誰が確認し、どう対応したか」
を本部と現場で同じ見方に揃えることを目指しています。

まずは貴社の施設構成に合わせて、どの施設から始めると検証しやすいかを短時間で相談させてください。
```

## 避ける表現

- 売上向上や稼働率改善を断定する表現
- 実予約や宿泊実績を把握しているように見える表現
- データ取得方法を前面に出す表現
- 競合施設を監視している印象が強すぎる表現
- AIが自動で最適価格を決めるように読める表現
- 既存PMSやサイトコントローラーを置き換えるように読める表現
- Revenue担当者が不要になるように読める表現
- 値下げ支援ツールに見える訴求
