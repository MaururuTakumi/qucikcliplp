# エスカレーション方針

## 自走してよい
既存コンポーネントの合成 / セクション・ページ追加 / 正データ準拠のコピー流し込み / inverse適用 / ArrowCTA・Reveal配置 / 監査対応 / Tailwind `ds-*` でのスタイリング。

## fableに回す（仕様に落ちない"手触り"・発明系）
- 3D（embraceStack #19 等）・新モーションプリミティブの発明。
- デザイン言語レベルの判断（新しい色体系・タイポ設計・レイアウト原則の新設）。
- 「LayerX級の質感」を出すための微調整で、仕様化されていないもの。
- 既存デザインシステムの規約自体を変える提案。

## 依頼の出し方（fableへ）
- 何を・なぜ自分で決められないか（＝仕様に落ちない手触り/発明だから）を1-2文で。
- 対象issue番号 or 対象ファイル、期待する成果物（設計図 or craft実装）を明記。
- 例:「#19 embraceStackの3D造形は仕様化されていない手触りなのでfableへ。progress振り付けの最終値と材質パラメータの設計を依頼」。
